The data is saved in the TRACK database table:

return_shipping_cost_analytics

It is also exported from sequelize.js:144 as ReturnShippingCostAnalytics.

Example service/query:

const { ReturnShippingCostAnalytics } = require('../sequelize');

async function getReturnShippingCostRows() {
return ReturnShippingCostAnalytics.findAll({
    where: { category: 'Refund' },
    order: [
    ['country', 'ASC'],
    ['intervalStartKg', 'ASC'],
    ],
    raw: true,
});
}

Useful fields for a view-only table/graph:

- country
- currency
- weightInterval such as 0-1 kg
- entryCount
- p95AmountDisplay for display, or p95Amount for chart values
- guardrail such as Raised from 2,400 JPY
- generatedAt

Each time the Return shipping cost analytics tool runs, it deletes the old Refund rows and inserts the newly generated rows. So the table represents the latest generated snapshot, not historical runs.
