I want to remove the current "Case Tracker" (routes/ct.js) from this project, and replace with a much simplified tool that's just a single database table with a simple/quick to use UI.

Importantly, a case should track the following data:
* Order number: used as unique case id
* Customer id: used to track repeaters
* Customer complaint: selected from a pool of predefined values (can only be set once, relpaced by "Customer complaint edit" if changed)
* Customer complaint edit: same as "Customer complaint" if changed after created, and replaces "Customer complaint" if a value has been set (example "Lost complaint" -> "delivered" -> "Defect complaint")
* Customer complaint comment: optional extra details
* Shipping method: required for shipping related complaints (lost/stuck in shipment, shipping damages)
* Shipping date: required for lost/stuck in shipping complaints
* Complaint date: same date as added to database, or date provided by user
* Defect items: comma separated list of item codes, required for defect complaints
* Defect description: short description of the defect, required for defect complaints
* Solution: final solution for the case, from predefined list of solutions
* Solved date: date the case was closed

Replace the "Case Tracker" with this simplified tool, and create 3 database tables: "cases", "complaint types", "solution types".

The tool itself should have 2 pages, a landing page, with a order number input field + "open case" button, for either open an existing case (order number exist in database), or create a new case, and a list of open cases; and a view/edit case page, that display any existing data and a minimal form for editing the case.

Also add an admin page for adding/deleting "complaint types" and "solution types" database entries, and in the "cases" database, save the text values, so nothing break if any types are deleted.

I do plan to add an analytics page later, but first we need to start gather data, so just create a placeholder page, that list the total number of cases for now.
